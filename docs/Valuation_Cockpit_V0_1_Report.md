# Valuation Cockpit V0.1 Report

## 1. 本次功能范围

本次新增 Rachel Capital OS 本地估值工具 V0.1：Valuation Cockpit 标的识别与模型推荐引擎。

功能范围：

- 在 Streamlit 本地 Web App 新增 `估值驾驶舱` 页面。
- 支持用户输入标的基本信息。
- 自动识别标的类型。
- 推荐适合的估值模型。
- 展示多模型适用性对比表。
- 输出需要补充的数据清单。
- 支持生成 Obsidian 估值备忘录草稿。

本阶段不进行完整估值计算，不输出估值结论，不接真实交易系统，不接自动下单，也不发布到 GitHub Pages。

## 2. 五类标的分类体系

| 标的类型 | 定义 | 核心问题 |
| --- | --- | --- |
| 上市公司 | 已在 A股、港股、美股或其他公开市场上市交易的公司 | 当前市值是否处于合理区间 |
| 未上市成长公司 | 未上市，但具备完整公司主体、产品、团队、融资历史和长期成长空间的公司 | 公司整体合理估值和未来退出价值 |
| 一级市场融资标的 | 正在融资、老股转让、Pre-IPO、产业轮融资或其他一级市场交易的标的 | 本轮投前/投后估值是否合理 |
| 项目公司 / SPV | 为项目、工程、资产、园区、算力中心、电站、矿山或回收项目设立的公司或 SPV | 项目现金流、回收期和 IRR 是否合理 |
| 资产型项目 | 核心价值来自资产、资源、牌照、合同、产能、土地、电力指标、矿权或设备的项目 | 资产本身价值、稀缺性和现金流 |

## 3. 自动识别规则

当前使用简单规则引擎：

- 如果 `是否上市 = 是`，识别为 `上市公司`。
- 如果 `是否上市 = 否` 且 `完整公司主体 = 是`，识别为 `未上市成长公司`。
- 如果 `是否上市 = 否` 且 `正在融资或交易 = 是`，识别为 `一级市场融资标的`。
- 如果 `单一项目 / SPV = 是`，识别为 `项目公司 / SPV`。
- 如果 `主要依赖资产或资源 = 是`，识别为 `资产型项目`。
- 如果多个条件同时满足，系统输出主分类和辅助分类。

主分类优先级：

1. 项目公司 / SPV
2. 资产型项目
3. 上市公司
4. 一级市场融资标的
5. 未上市成长公司

该优先级用于处理项目型交易、资产型项目和一级市场交易重叠的场景。

## 4. 模型推荐规则

| 标的类型 | 主模型 | 辅助模型 |
| --- | --- | --- |
| 上市公司 | PE、PB、PS、EV/EBITDA、DCF、PEG、SOTP | 历史估值分位、同业比较、情景估值 |
| 未上市成长公司 | 可比上市公司法、可比融资交易法、收入倍数、ARR 倍数、SOTP | 用户价值法、流动性折扣、上市概率折扣、二级市场映射法、情景估值 |
| 一级市场融资标的 | 可比融资交易法、可比上市公司法、融资轮次估值锚、收入倍数、利润倍数、ARR 倍数 | 订单倍数、技术成熟度折扣、流动性折扣、退出路径折扣、下轮融资估值推演、IPO / 并购退出倒推 |
| 项目公司 / SPV | DCF、IRR、投资回收期、项目现金流模型 | 资产价值法、产能价值法、合同现金流法、政府补贴敏感性分析、利用率敏感性分析、融资结构敏感性分析 |
| 资产型项目 | 资产重估法、重置成本法、DCF、单位产能估值 | 资源储量估值、租金/合同现金流估值、折现回收期、可交易资产比较法 |

特殊规则：

- 如果收入增长状态为 `周期波动`，或标的名称包含 `船舶`，上市公司主模型优先展示 `PB`、`周期归一化 PE`、`EV/EBITDA`、`SOTP`。
- 如果没有收入，PE、PEG、PS、收入倍数会被降级为不适用。
- 如果现金流稳定，DCF 会被提升到辅助模型或主模型视角。

## 6. 当前限制

- V0.1 仅做标的识别和模型推荐，不计算完整估值。
- 尚未接入 Tushare、yfinance、Alpha Vantage、FRED 或公司财报数据库。
- 尚未自动读取 Obsidian 公司卡片。
- 尚未生成 Excel / CSV 模型。
- 尚未建立 DCF、IRR、SOTP 等模型的参数输入和计算引擎。
- 不输出任何操作结论。
- 不连接交易系统，不自动下单。
- 不发布到 GitHub Pages。
