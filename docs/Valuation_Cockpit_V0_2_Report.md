# Valuation Cockpit V0.2 Report

## 1. 用户反馈

V0.1 将已上市公司、未上市成长公司、一级市场融资标的、项目公司 / SPV 和资产型项目放在同一个输入表单中，导致页面复杂、字段混杂，用户难以快速判断应该填写哪些信息。

本次重构目标是把公开市场估值和非公开市场估值拆成两个独立板块。

## 2. 为什么要拆分已上市与未上市估值

已上市公司和未上市 / 一级市场标的的核心问题不同：

- 已上市公司：公开市场当前市值是否处于合理区间。
- 未上市 / 一级市场：非公开市场、融资交易或项目投资中的合理估值框架是什么。

两类标的需要的数据、估值模型、折扣因素和输出结构不同。如果混在同一张表单中，会让上市公司用户看到项目字段，也会让项目/SPV 用户看到 PE、PB、PEG 等不一定适用的字段。

## 3. 新页面结构

页面：`streamlit_app/pages/2_Valuation.py`

页面标题：`Rachel 估值驾驶舱`

页面内拆为两个 Tab：

1. 已上市公司估值
2. 未上市 / 一级市场估值

每个 Tab 只展示与该板块相关的字段，并在输出区展示模型、数据清单、风险和 Obsidian 备忘录生成按钮。

## 4. 已上市公司估值逻辑

适用范围：

- A股上市公司
- 港股上市公司
- 美股上市公司
- 港股通标的
- 已上市但存在周期性、成长性、资产重估、SOTP 等不同估值逻辑的公司

输入字段：

- 股票名称、股票代码、市场、所属行业、战略生态
- 是否盈利
- 收入增长状态
- 利润增长状态
- 资产属性
- 现金流稳定性
- 是否强周期
- 是否适合 SOTP
- 是否处于困境反转

模型逻辑：

| 公司画像 | 主模型 |
| --- | --- |
| 稳定盈利公司 | PE、PEG、DCF、EV/EBITDA、历史估值分位、同业比较 |
| 高成长但盈利波动公司 | PS、EV/Sales、远期 PE、PEG、情景估值、同业比较 |
| 强周期公司 | PB、周期归一化PE、EV/EBITDA、历史周期分位、资产重估法 |
| 重资产制造公司 | PB、EV/EBITDA、ROE-PB、订单利润法、DCF 辅助 |
| 平台型公司 | SOTP、PE、PS、DCF、用户价值法、同业比较 |
| 资源型公司 | PB、EV/EBITDA、资源储量估值、周期归一化利润、资产重估法 |

输出：

- 标的画像
- 系统推荐估值模型
- 多模型适用性对比表
- 当前需要补充的数据清单
- 初步估值框架
- 风险与不确定性
- 生成 Obsidian 估值备忘录草稿按钮

## 5. 未上市 / 一级市场估值逻辑

适用范围：

- 未上市成长公司
- 一级市场融资标的
- Pre-IPO 标的
- 老股转让标的
- 项目公司 / SPV
- 资产型项目

内部分四类：

1. 未上市成长公司
2. 一级市场融资标的
3. 项目公司 / SPV
4. 资产型项目

自动识别规则：

- 完整公司主体 = 是，正在融资 = 否，单一项目/SPV = 否：未上市成长公司。
- 正在融资或老股转让 = 是：一级市场融资标的。
- 单一项目 / SPV = 是：项目公司 / SPV。
- 主要依赖资产、资源、牌照或合同 = 是：资产型项目。
- 多个条件同时满足时，输出主分类、辅助分类和判断理由。

模型逻辑：

| 类型 | 主模型 | 必须考虑 |
| --- | --- | --- |
| 未上市成长公司 | 可比上市公司法、可比融资交易法、收入倍数、ARR 倍数、用户价值法、SOTP、二级市场映射法 | 流动性折扣、上市概率折扣、信息透明度折扣 |
| 一级市场融资标的 | 可比融资交易法、可比上市公司法、融资轮次估值锚、收入倍数、利润倍数、ARR 倍数、订单倍数、IPO / 并购退出倒推 | 技术成熟度折扣、流动性折扣、退出路径折扣、下一轮融资估值推演 |
| 项目公司 / SPV | DCF、IRR、投资回收期、项目现金流模型、合同现金流法、产能价值法 | 利用率敏感性、融资结构敏感性、政府补贴敏感性、关键合同风险 |
| 资产型项目 | 资产重估法、重置成本法、DCF、单位产能估值、资源储量估值、租金 / 合同现金流估值、可交易资产比较法 | 资产可交易性、政策限制、现金流稳定性、折旧与维护成本 |

输出：

- 标的画像
- 主分类与辅助分类
- 判断理由
- 推荐估值模型
- 多模型适用性对比表
- 必须加入的折扣或敏感性因素
- 当前需要补充的数据清单
- 初步估值框架
- 风险与不确定性
- 生成 Obsidian 估值备忘录草稿按钮

## 7. 当前限制

- V0.2 仍然只做标的画像识别和估值框架推荐，不做完整估值计算。
- 尚未接入实时行情、财报、Tushare、yfinance、Alpha Vantage 或 FRED。
- 尚未自动读取 Obsidian 公司卡片或项目文件。
- 尚未生成 Excel 模型。
- 项目/SPV 的 IRR、回收期、DCF 仍未进入数值计算阶段。
- 不接交易系统，不自动下单。
- 不发布到 GitHub Pages。
